# Mac Keys for Windows

Мини-утилита для Windows, которая делает хоткеи как на маке. Висит в трее,
стартует при входе в систему, полностью настраивается из конфига.

Маппинг идёт **по физическому положению клавиш** рядом со спейсом:

| Mac            | Windows       |
|----------------|---------------|
| `Cmd`          | `Alt`         |
| `Option`       | `Win`         |
| `Ctrl`         | `Ctrl`        |

То есть на Windows ты жмёшь `Alt+C` — утилита отправляет `Ctrl+C`, и в любой
программе это копирует. То же самое для save, paste, find, new tab, скриншотов,
переходов в начало/конец строки и т.д.

## Установка

### Вариант 1 — готовый exe (рекомендуется)

1. Скачай `mac-keys-for-windows.zip` со страницы Releases и распакуй в любую
   папку (например, `C:\Tools\mac-keys-for-windows\`).
2. ПКМ по `install.ps1` → **Run with PowerShell**. Если Windows ругается на
   execution policy — открой PowerShell и выполни:
   ```powershell
   powershell -ExecutionPolicy Bypass -File .\install.ps1
   ```
3. При первом запуске появится диалог: **«Включить автозапуск при входе в
   Windows?»** — выбираешь Yes/No. Yes регистрирует Scheduled Task (UAC
   спросит подтверждение), No — просто работает в текущей сессии. Включить
   позже можно из окна настроек (чекбокс **Run at Windows logon**) или из
   трея (**Run on startup**).
4. В трее появится цветная иконка: **зелёная** = активна, **жёлтая** =
   пауза, **красная** = выключена.

### Вариант 2 — из исходников

1. Поставь [AutoHotkey v2](https://www.autohotkey.com/).
2. Клонируй репозиторий в удобное место.
3. Запусти `install.ps1` тем же способом — скрипт сам найдёт AHK и
   зарегистрирует задачу, указывающую на `mac-keys.ahk`.

### Удаление

Запусти `uninstall.ps1`. Задача автозапуска удалится, процесс остановится.
Config и лог в `%APPDATA%\mac-keys-for-windows\` останутся — удалишь вручную,
если не нужны.

## Как пользоваться

**Левый клик по иконке** — открывает мини-окно настроек:

- статус (Active / Paused / Disabled, цветной кружок),
- три чекбокса: **Enabled (master)**, **Auto-pause in fullscreen apps**,
  **Smart gating**, **Run at Windows logon**,
- два списка: **Excluded apps** и **Fullscreen whitelist** с кнопками
  «Add current app» / «Remove selected»,
- внизу: Edit config / Reload / Log / Quit.

**Глобальные хоткеи:**

- `Ctrl+Alt+Shift+M` — master toggle (включить/выключить).
- `Ctrl+Alt+Shift+Q` — kill-switch, полностью завершает утилиту.

**Правый клик по иконке** — обычное tray-меню:

- **Включено** — master-переключатель. Продублирован хоткеем
  `Ctrl+Alt+Shift+M` (меняется в конфиге).
- **Пауза на 30 секунд / Снять паузу** — временно отдаёт Alt обратно Windows
  без полного отключения.
- **Отключить в текущем приложении** — добавляет активный процесс в
  `excludedApps`, утилита игнорит его целиком.
- **Держать активным в текущем приложении (fullscreen)** — добавляет в
  `fullscreenWhitelist`. Нужно, если приложение уходит в F11/fullscreen, но ты
  хочешь, чтобы хоткеи там продолжали работать (например, браузер, видеоплеер).
- **Автопауза в fullscreen** — галка включает/выключает авто-паузу, когда на
  экране процесс **не** в whitelist и он на весь экран (эвристика под игры).
- **Открыть config.json** — откроет файл в notepad. Сохраняешь — утилита
  автоматически перезагружается, изменения применяются.
- **Запуск при входе в систему** — галка регистрирует/снимает Scheduled Task.

## Как ловим игры

В игре `Alt` — это часто нативная клавиша (крауч, войс, walk). Если просто
вешать `Alt+C → Ctrl+C` глобально, игры ломаются. Защита идёт в пять слоёв,
**без хардкод-листа имён игр**:

1. **Smart gating (универсальный).** В каждый момент времени утилита знает,
   как давно зажат Alt и были ли нажатия WASD во время удержания.
   - `< 250 мс` зажатого Alt → это шорткат, отрабатываем (`Alt+C → Ctrl+C`).
   - `> 800 мс` или WASD во время удержания → это game-key, **пропускаем
     нативный Alt в игру** (отправляется `{Blind}c` без модификатора).
   - Серая зона (250–800 мс без WASD) — решается по движению.
   Пороги настраиваются в `config.smartGating`.
2. **Авто-пауза в fullscreen.** Утилита проверяет окно в фокусе: если оно
   занимает весь монитор и у него нет title bar → пауза. Покрывает большинство
   игр (и exclusive, и borderless fullscreen).
3. **Whitelist процессов, которые можно игнорить**: браузеры, видеоплееры,
   IDE, офис. В них fullscreen не триггерит паузу.
4. **Blacklist процессов**, где утилита не работает никогда: RDP, VMware,
   VirtualBox, Parsec, AnyDesk, TeamViewer. Ввод уходит насквозь.
5. **Master-toggle и kill-switch.** `Ctrl+Alt+Shift+M` — пауза/работа,
   `Ctrl+Alt+Shift+Q` — полностью завершить утилиту.

Если игра всё же пропустилась (например, оконный режим + нестандартное окно) —
открыл окно настроек → «Add current app» в Excluded apps. Один клик.

## Конфиг

Файл: `%APPDATA%\mac-keys-for-windows\config.json`.

```json
{
  "enabled": true,
  "autoPauseFullscreen": true,
  "smartGating": {
    "enabled": true,
    "altShortcutMaxMs": 250,
    "altGameMinMs": 800
  },
  "masterToggleHotkey": "^!+m",
  "quitHotkey": "^!+q",
  "excludedApps": ["mstsc.exe", "parsec.exe"],
  "fullscreenWhitelist": ["chrome.exe", "vlc.exe"],
  "bindings": [
    { "from": "!c", "to": "^c", "enabled": true, "desc": "copy" },
    { "from": "!+4", "to": "#+s", "enabled": true, "desc": "screenshot region" }
  ]
}
```

### Нотация хоткеев (AutoHotkey v2)

| Символ | Что значит   |
|--------|--------------|
| `^`    | Ctrl         |
| `!`    | Alt          |
| `+`    | Shift        |
| `#`    | Win          |
| `{F4}` | именованная клавиша |
| `Space`, `Left`, `Right`, `Up`, `Down`, `Home`, `End`, `BackSpace`, `PrintScreen` | как есть |

Примеры:
- `!c` = `Alt+C`
- `!+4` = `Alt+Shift+4`
- `!{F4}` = `Alt+F4`
- `#+s` = `Win+Shift+S`

После правки конфига ничего перезапускать не надо — утилита детектит
изменение файла и сама перезагружается за 2 секунды.

## Что по умолчанию замаплено

- **Редактирование:** `Alt+C/V/X/A/Z/Shift+Z/S/Shift+S/W/Q/T/Shift+T/N/Shift+N/O/P/R/F/G/Shift+G/B/I/U/K/L`
- **Вкладки:** `Alt+Shift+]` / `Alt+Shift+[` — следующая/предыдущая вкладка
- **Скриншоты:** `Alt+Shift+3` (fullscreen), `Alt+Shift+4` и `Alt+Shift+2`
  (region), `Alt+Shift+5` (запись через Game Bar)
- **Поиск:** `Alt+Space` → `Win+S` (системный поиск)
- **Навигация:** `Alt+←/→/↑/↓` — начало/конец строки, верх/низ документа;
  `Alt+Shift+←/→/↑/↓` — то же с выделением
- **По словам (Option+←/→):** `Win+←/→` — перемещение, `Win+Shift+←/→` —
  с выделением, `Win+Backspace` — удалить слово назад
- **Backspace:** `Alt+Backspace` — удалить слово (как на маке)

Всё редактируется в `config.json`. Любой пункт можно выключить флагом
`"enabled": false`.

## Важные оговорки

- **Alt+F открывает "Найти", а не меню File.** Это плата за маковский
  маппинг. Доступ к меню: `F10` или мышкой. Если сильно мешает — в конфиге
  выстави `"enabled": false` у биндинга `!f`, меню File вернётся.
- **Alt+Space** перехватывается для системного поиска, поэтому нативное
  системное меню окна (переместить/свернуть) через `Alt+Space` больше не
  доступно. Используй кнопки в заголовке или `Win+стрелка`.
- **Utility работает только пока процесс запущен.** Это не драйвер уровня
  ядра — если выйти через трей, всё вернётся к нативному поведению Windows.
- **Автозапуск опционален и явный.** При первом запуске спрашивается отдельным
  диалогом, никакой тихой регистрации. Если согласишься — UAC отдельно
  попросит подтверждение на создание Scheduled Task с HighestAvailable.
  Админские права нужны, чтобы хоткеи работали в elevated-окнах (Task Manager,
  regedit). Без автозапуска утилита просто работает в текущей сессии.

## Лог и отладка

`%APPDATA%\mac-keys-for-windows\mac-keys.log` — простой текстовый лог (старт,
ошибки биндингов, перезагрузки конфига).

Если хоткей не отрабатывает:
1. Проверь лог — не было ли ошибки регистрации.
2. Проверь, не попало ли приложение в `excludedApps`.
3. Убедись, что утилита не на паузе (иконка в трее).
4. Если приложение elevated (admin) — нужно, чтобы утилита тоже была elevated
   (через Scheduled Task как раз решается).
